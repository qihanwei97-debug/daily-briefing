### 🔥 头条一：Anthropic 与 OpenAI 同日宣布 PE 巨头加持的企业 AI 合资公司——总承诺金额 $11.5B、矛头直指传统咨询业

5 月 4 日（周一）AI 行业出现一个反复多年传言后终于落地的剧本：**Anthropic 与 OpenAI 在同一天分别宣布与华尔街最大 PE / 投行联手组建独立企业 AI 服务公司**，业务模式均为"把模型 + 工程师嵌入 PE 投后中型企业，重写工作流并把 AI 整合进核心运营"，与麦肯锡 / BCG / Accenture / Deloitte 这类咨询厂商形成正面竞争。

**Anthropic 阵营（$1.5B 总承诺）：** Anthropic、Blackstone、Hellman & Friedman 各出资约 $300M、Goldman Sachs 出资约 $150M，外加 Apollo Global Management、General Atlantic、Leonard Green、GIC、Sequoia Capital 组成的二级资助财团（[Anthropic 官方公告（fetch 403）](https://www.anthropic.com/news/enterprise-ai-services-company) | [Blackstone 公告（fetch 403）](https://www.blackstone.com/news/press/anthropic-partners-with-blackstone-hellman-friedman-and-goldman-sachs-to-launch-enterprise-ai-services-firm/) | [CNBC](https://www.cnbc.com/2026/05/04/anthropic-goldman-blackstone-ai-venture.html) | [Fortune 矛头指向咨询业](https://fortune.com/2026/05/04/anthropic-claude-consulting-industry-joint-venture-blackstone-goldman-sachs/) | [Bloomberg](https://www.bloomberg.com/news/articles/2026-05-04/goldman-blackstone-partner-with-anthropic-on-ai-services-firm) | [GIC 公告](https://www.gic.com.sg/newsroom/all/anthropic-partners-with-blackstone-hellman-friedman-and-goldman-sachs-to-launch-enterprise-ai-services-firm/) | [Reuters/Investing.com](https://www.investing.com/news/stock-market-news/anthropic-nears-15-billion-ai-joint-venture-with-wall-street-firms-wsj-reports-4654946)）。新公司将派 Anthropic 的 applied AI 工程师驻进客户内部识别 Claude 落点、定制方案并提供长期支持——按 Fortune 5/4 措辞"不像传统咨询，更像把工程师嵌入企业重塑流程"。

**OpenAI 阵营（$10B 估值，$4B+ 已到位）：** "The Deployment Company"——TPG 领投，Brookfield Asset Management、Advent、Bain Capital 等共 19 家投资方；OpenAI 自身承诺最高 $1.5B（首期 $500M 股权 + 后续可加 $1B），保留 super-voting shares 维持战略控制（[Bloomberg](https://www.bloomberg.com/news/articles/2026-05-04/openai-finalizes-10-billion-joint-venture-with-pe-firms-to-deploy-ai) | [TheNextWeb 完整结构](https://thenextweb.com/news/openai-deployco-finalized-10-billion-joint-venture) | [TechCrunch 双 JV 对比](https://techcrunch.com/2026/05/04/anthropic-and-openai-are-both-launching-joint-ventures-for-enterprise-ai-services/) | [Axios](https://www.axios.com/2026/05/04/openai-anthropic-private-equity-enterprise-business) | [Technobezz 17.5% 详解](https://www.technobezz.com/news/openai-guarantees-175-percent-annual-return-in-unusual-10-billion-enterprise-deal)）。**关键非常规条款：OpenAI 向 PE 出资方承诺 5 年期 17.5% 年化保底回报**——Technobezz 与 TheNextWeb 均指出这在 PE 圈"不常见"，运营方主动对财务投资人写出年化回报承诺，相当于 OpenAI 用未来现金流为 PE 资金做保底。优先服务行业为医疗、物流、制造、金融服务，按 Palantir "forward-deployed engineer" 模式派工程师驻场。

**为什么重要：** 第一，**两家同日宣布**——意味着 OpenAI 与 Anthropic 已就"PE 投后通道是 2026 年企业 AI 落地的最佳分销路径"达成战略一致——传统企业软件销售周期被两家共同判定为太慢、捕获不到 2026 年的需求峰值。第二，**Anthropic 与 OpenAI 同时进入"AI + 咨询"赛道，等于直接威胁麦肯锡/BCG/Deloitte 等传统咨询业的 AI 转型咨询业务**——按 Fortune 5/4 标题措辞"Anthropic 向咨询业开了一枪"。第三，**OpenAI 的 17.5% 保底回报条款**——意味着 OpenAI 资产负债表上将出现一笔**结构性次级负债（structurally subordinated paper）**——若 The Deployment Company 5 年内未达到 17.5% 年化回报，差额由 OpenAI 兜底；这是 OpenAI 接近 IPO 窗口前的非常规财务安排。第四，**两家共选 PE 渠道而非传统 SI**——意味着资金、客户、渠道均通过"封闭式 PE 网络"快速触达，不必走传统企业销售线；同时 PE 自身希望通过 AI 提升投后估值（exit valuation），形成"双方都赢"闭环。

- 📍 获取（PE 投后中型企业为主，需通过对应 PE 关系切入）：Anthropic 阵营客户由 Blackstone / H&F / Goldman 等 PE 投后池开始；OpenAI 阵营客户由 TPG / Brookfield / Advent / Bain 投后池开始
- 💡 上手参考：律所代表 Big4 / 麦肯锡类咨询业客户的同事可关注未来 6-12 个月"AI 咨询业务被 hyperscaler 直接抢占"对客户营收结构的影响；代表 PE 客户的同事可关注合资公司服务条款（如数据使用、IP 归属、客户独占性）的范本

## ⑤ 本期技巧：用 Claude Code v2.1.128 新增的 `/recap` 重启被打断的会话——而不是从头复述上下文

**场景：** 你早上在用 Claude Code 处理一份 SPA 的修订条款比对——读了 60 页对照文本、让 Claude 列出 12 处疑点、又对其中 5 处做了进一步深挖；正写到一半时电话进来要先处理另一件紧急事，等 3 小时后再回到 SPA 这件事时，Claude 这边的上下文你大致还记得，但具体走到哪一步、上一轮指令是问的什么、下一步本来打算让它做什么——已经模糊。常见的做法是手动翻聊天记录（费时）或简单粗暴重新发一遍 prompt（浪费已耗的 token + Claude 可能换个角度重出结果，连贯性差）。这正是 Claude Code v2.1.128 新增的 `/recap` 命令要解决的问题。

**为什么好用：** `/recap` 是 Anthropic 官方在 2.1 系列中明确为"长会话回归"设计的辅助命令——按 5/4 raw CHANGELOG 措辞，它会"展示无焦点期间发生了什么"（"shows what happened during unfocused time"）。本质上它把当前会话的**进展节点**抽取出来形成一份摘要，告诉你："上一次你让我做的是 X、我返回的是 Y、目前未完成的是 Z"——读完这份摘要再继续，比从头翻聊天记录效率高一个量级，比"重新发一遍 prompt"更准确（避免 Claude 再走一遍同样的推理路径）。比起手写 README 维护项目状态，`/recap` 的优势是它从对话日志直接抽取不需手维护，也不会漏。

**操作路径：**

1. **回到被搁置的会话后，先执行 `/recap`（不带参数）**——Claude Code 会输出一段摘要，包含"上次你问什么 / 我回答什么 / 当前进度位置"。读完这段你就知道上次走到哪了。
2. **如果想把 `/recap` 变成默认行为**——执行 `/config`，找到 recap 相关选项打开"自动 recap on session resume"——下次重开会话时会自动先跑一遍 recap，不用手动调用。
3. **配合 `/color` 使用——给每个客户矩阵分配固定主题色**（v2.1.128 新增）：典型用法是 `/color <颜色名>`，例如 `red` 给紧急客户、`blue` 给 SPA 项目、`green` 给日常事务——视觉上一眼区分当前会话属于哪个客户，避免误把 A 客户合同贴到 B 客户的会话里。`/color` 不带参数则随机分配——多窗口同时开时也能立刻区分。
4. **结合 v2.1.121 引入的 `claude plugin prune`**——长会话往往会累计一堆已不需要的插件，定期 prune 一下避免 MCP 服务连接过多导致响应变慢。

**注意事项：** 第一，**`/recap` 摘要并不能完全替代你自己读一眼最近 10-20 轮对话**——尤其当涉及客户具体合同条款或敏感数据时，摘要可能漏关键细节，最稳的做法是先 `/recap` 看大方向、再翻最后 5-10 轮确认具体上下文；第二，**`/recap` 在 token 上有成本**——它需要重读会话历史生成摘要，在长会话中可能消耗较多 token；如果会话已超过 200k tokens 上下文，可考虑先 `/compact` 压缩再 `/recap`；第三，**`/recap` 不会自动保存到磁盘**——若需要跨日跨设备的"项目状态档"，仍应在 `CLAUDE.md` 或客户文件夹的 `STATUS.md` 中手动记录关键决策，`/recap` 只解决单次会话内的回归问题，不替代项目级文档；第四，**对涉客户敏感信息的会话，`/recap` 输出本身也含敏感数据**——若复制粘贴用于沟通，需自查是否含客户姓名、合同金额、个人信息等。
