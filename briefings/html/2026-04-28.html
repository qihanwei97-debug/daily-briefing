Claude Code `/team-onboarding` 把"老律师的 AI 工作流"打包给新人

场景： 你是汉坤一位习惯用 Claude Code 写法律备忘录的合伙人，过去三个月已经把"把客户长合同丢进 CLAUDE.md → 用 `/init` 建项目骨架 → 用自定义 skill 跑 citation cross-check → 用 MCP 连接威科先行检索"这套流程跑得顺手，准确率和效率都比手工高一截。新晋律师 Lucy 4 月 28 日刚入职，她的 Macbook 装好了 Claude Code，但面对你贡献到 git 的 30 多个项目目录，她不知道哪一个是"主力"，也不知道你常用哪些斜杠命令、哪些 MCP server、哪些 prompt 模板。过去通常的做法是你抽两小时一对一带教——但 Anthropic 4 月 10 日上线的 `/team-onboarding` 命令把这个时间成本基本降到零。

Claude Code 官方 Week 15 摘要（v2.1.101 上线） | Smartscope 上手指南 | Joe Njenga 实测笔记 | Apiyi 中文整理

为什么好用： Claude Code v2.1.101 起内置的 `/team-onboarding` 命令会扫描你最近 30 天的 Claude Code 使用记录——所有会话、命令、MCP server 调用、subagent 调用、自定义 skill 触发，自动生成一份可读的 Markdown 入职指南，新人拿过来就能复现你的工作环境。它的根本价值在于"知识传承不再依赖手写文档"——你怎么用 Claude Code 写东西，它就把"怎么用"反向工程出来。律所内部最稀缺的就是"把高级合伙人的隐性工作流显性化"的能力，这条命令直接戳中。

操作路径（4 步）：

  1. 在你的主力项目目录运行命令——必须是你最近 30 天真实用过 Claude Code 的项目（首次用、空 30 天的项目跑出来是空指南）： ``` cd ~/projects/汉坤合同主力库 claude > /team-onboarding ```
  2. 审阅生成的 onboarding 指南——Claude Code 会输出一份 Markdown 文件，包含你常用的斜杠命令、MCP 服务器列表、subagent 配置、典型 prompt 模板，以及推荐的 CLAUDE.md 内容。先自己读一遍删掉敏感信息（具体客户名、未公开案号、内部 prompt 中的法律观点等）
  3. 沉淀到 CLAUDE.md + 项目 settings——按官方文档建议（Claude Code Commands 参考），把生成指南中"稳定的规则"挑出来固化到项目根目录的 `CLAUDE.md`，把"团队共享的 MCP 配置"放进 `.claude/settings.json` 并通过 git 共享
  4. 把指南交给新人——Lucy 拿到这份指南后，按文中"项目结构-常用命令-MCP 配置"三段，30 分钟内就能跑出第一份合规的客户备忘录初稿

注意事项： ① 千万先脱敏——指南会包含你过去 30 天的具体 prompt 措辞，里面可能含客户身份、案件细节、内部法律意见，公开前必须人工过一遍；② `/team-onboarding` 不是 CLAUDE.md 替代品——它是"我现在怎么用"的快照，不是"应该怎么用"的规范；正确做法是用它发现稳定模式 → 提炼成 CLAUDE.md 规则；③ 30 天窗口的局限——如果你最近一个月没有真正深度用 Claude Code 处理本类项目，生成出的指南会偏单薄，建议先自己跑两周再传承。
