头条一：OpenAI 5/7 上线 gpt-realtime 三件套——GPT-5 级推理语音 + 70 语种实时翻译

5 月 7 日（周四），OpenAI 通过 Realtime API 同时上线三个语音/音频模型——`gpt-realtime-2`、`gpt-realtime-translate`、`gpt-realtime-whisper`，覆盖"对话推理 + 跨语种翻译 + 转录"全工作流9to5Mac 5/7 | The Decoder 5/7 | Testing Catalog | Inc 5/7 | MarketScreener | Shopifreaks 详解 | OpenAI Realtime Translation Cookbook | OpenAI 官方介绍页（fetch 403）。OpenAI 一手介绍页今日 fetch 403，未取得官方一手内容；下文据 6 家英文媒体交叉报道还原。

核心数据（来源：OpenAI 对各家媒体披露 + Realtime Translation Cookbook 一手页）：

模型能力定价
`gpt-realtime-2`OpenAI 自评提供 GPT-5 级推理（GPT-5 系列在复杂指令、工具调用、上下文管理的能力等级）于实时语音对话；支持复杂请求处理、工具调用、上下文维护、纠错回滚$32 / M audio input tokens；$0.40 / M cached input；$64 / M audio output
`gpt-realtime-translate`实时语音 → 语音翻译 + 文字转录，自动检测源语言；支持 70+ 输入语言、13 个输出语言；广播、流媒体、电话、视频会议$0.034 / 分钟
`gpt-realtime-whisper`实时语音 → 文本转录（caption / 会议纪要 / workflow updates）$0.017 / 分钟

早期客户（OpenAI 官方点名）： - Zillow——美国房地产经纪平台，用于电话语音 agent；据 Air Street Press 5 月报告引用 OpenAI 数字，通话成功率提升 26 个百分点 - Priceline——在线旅游代理，用于多语种旅行助理 - Deutsche Telekom——欧洲电信运营商，用于多语言客户支持

为什么重要： 第一，这是 OpenAI 首次将 GPT-5 系列的推理能力搬进 Realtime API——此前 Realtime API 用的是 gpt-4o-realtime 系列，性能停留在调用-响应级别；`gpt-realtime-2` 在多步推理、工具调用、上下文记忆三方面跨级，令"语音 agent 直接处理复杂事务"从概念跨到工程；Zillow 26 个百分点的电话通话成功率提升是这条能力跃迁的首个外部硬数据。第二，翻译模型与转录模型分拆定价——$0.034/分钟翻译 + $0.017/分钟转录的"按分钟计费"是 OpenAI 在 Realtime 系列首次脱离 token 计费、转向更贴近电信/SaaS 客户预算逻辑的定价；70 输入 / 13 输出的非对称设计，反映当前主流跨境业务流向（多源说英语/普通话/西班牙语等少数主流语 → 单点服务商）。第三，与 Anthropic Claude Voice、Google Gemini Live、ElevenLabs Conversational AI 的赛道之争进入决战——Realtime API 的开放生态意味着 SaaS 厂家和电信运营商不再被锁定在单家厂商；定价 $32/M audio input vs Anthropic 当前未公开 Voice API 定价、vs ElevenLabs Conversational AI 高得多的 per-minute 费率，OpenAI 将企业语音侧的"性价比标尺"重置。第四，对中国市场的外溢——70 输入语言含主要东亚语种，但 13 输出语言以英语 / 西班牙语 / 法语 / 德语等欧美主流语为主，未公开是否含中文输出——出海客服 / 跨境电商客户在 5-6 月可能基于实测决定切换。

头条二：Claude Managed Agents 5/7 三大升级——Dreaming + Outcomes + Multiagent Orchestration

5 月 7 日（周四），Anthropic 在 Claude Managed Agents 平台同步推出三项关键能力——Dreaming（agent 之间的睡眠态记忆整理）、Outcomes（rubric 驱动的自评-重做闭环）、Multiagent Orchestration（lead agent 拆任务 + 多 specialist 并行）；同时为开发者开放 webhooksSD Times 5/7 | 9to5Mac 5/7 | The Decoder Dreaming 解析 | Digital Trends 5/7 | SiliconANGLE 5/6 | Android Authority | Claude API Docs - Dreams（fetch 通过，已采） | Anthropic 官方 blog（fetch 403）。Anthropic blog 一手页今日 fetch 403；但 Claude API Docs - Dreams 一手技术页 fetch 通过，下文 API 细节已据该页核实。

关键人物归因（R4）： Anthropic 在 5/7 官方 blog 中以 "agents that learn" 作为本批升级的核心论述（标题 "New in Claude Managed Agents: dreaming, outcomes, and multiagent orchestration"）；具体设计哲学的归属作者在媒体二手报道中未单列——Anthropic blog 一手页 fetch 403，未能取得作者署名。

为什么重要： 第一，Dreaming 是首个商用 LLM Agent 的"睡眠态记忆整理"机制——长期跑的 agent 在每次 session 写 memory store 时都是局部、增量的，累计后会出现重复、矛盾、过时条目；Dreaming 把"agent 在两次 session 之间用模型自身重新整理记忆库"作为可调用 API（async job、不修改输入、可弃用输出）暴露——对企业级 agent 长期运行（数月至数年）的可用性是质变。第二，Outcomes 把"自评 + 重做"工程化——过去 prompt-engineer 手写"reflect on your output and improve" 是松散且不可观测的；rubric + 独立 grader（隔离 context window）+ 强制反馈循环是把 RL-style 反馈搬进推理时；行业对比：OpenAI Agents SDK 5 月初有类似但更轻的"verifier"机制，而 Outcomes 把 grader 与 agent 完全隔离 context 是 Anthropic 的差异化。第三，Multiagent Orchestration 落地等于把"集体决策"产品化——lead-agent + specialist + 共享 filesystem 这套架构在 GitHub 开源生态已经有 `ruflo`、`crewAI`、Microsoft AutoGen 多家版本，Anthropic 把它做成第一方 API 而非 SDK 包意味着稳定性 / 可观测性 / 计费整合到 Claude 平台一层；与 5/7 同期 GitHub Trending #1 的 `ruflo`（45,655 stars）形成"商业第一方 vs 开源社区"双轨竞争。第四，对客户合规与审计的影响——multi-agent / dreaming 都涉及"agent 之间共享 / 整理工作记录"，这意味着客户合规线（GDPR / PIPL / HIPAA / 律师-客户特权 / 商业秘密）必须重新评估"哪些数据在 specialist 之间可以传递、哪些不行"；Outcomes 的 rubric 也成为可被原告 / 监管机构索取的"AI 决策标准记录"。
