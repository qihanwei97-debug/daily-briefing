## ⑤ 本期技巧：用 Claude Code Skills 把"判例引用核查"变成所内一键命令

**场景：** 你接到一份并购争议法律备忘录初稿，需要在交付前核对 18 条引用判例的真实性——案号是否存在、法院是否正确、引文页码是否被篡改。Sullivan & Cromwell 4/18 因 AI 幻觉申请规避制裁、Nebraska 律师 Greg Lake 4/15 被无限期吊销执照（详见 4/27 上期），你不会再让 Claude 直接生成 cite。但每份文书人工核 18 条又非常耗时，平均一名律师要花 2-3 小时手翻 Westlaw。理想状态是：所内每个律师在 Claude Code 里直接敲一行 `/cite-check 备忘录.md`，Claude 就能输出"哪几条 cite 在 Westlaw/北大法宝/LexisNexis 中确认存在、哪几条疑似幻觉、哪几条引文页码与原判决不符"的结构化报告。

**为什么好用：** Claude Code 在 4 月份引入了 **Skills 机制**——本质是一个把"复杂工作流封装为 slash command"的扩展系统。GitHub 上 andrej-karpathy-skills 仓库本周获 +44K 星，社区已开源大量通用 skill 模板。和"写一个 prompt 反复粘贴"相比，skill 的优势在于：① **跨会话复用**——一次定义、所内每个律师电脑都能调用；② **多步骤编排**——可以把"读取 .md → 提取 cite → 调外部工具核查 → 输出 markdown 报告"串成一个原子动作；③ **明确权限控制**——通过 `.claude/settings.json` 可限制 skill 只能调用预批准的 MCP 工具（Westlaw API、内部知识库），降低数据外泄风险。

**操作路径：**

1. **创建 skill 文件**：在所内 git 仓库（共享给全所律师）的 `.claude/skills/cite-check.md` 路径下，写一个 markdown 文件——开头用 YAML frontmatter 声明 `name: cite-check` 和 `description: 核查法律文书中的判例引用真实性`，正文写清楚"读取用户传入的文档、提取所有 `[Smith v. Jones, 123 F.3d 456 (2d Cir. 2019)]` 格式的 cite、对每条 cite 调 Westlaw MCP 工具查询是否存在、输出表格"的步骤指令。
2. **配置 Westlaw / 北大法宝 MCP 服务器**：在 `.claude/settings.json` 的 `mcpServers` 里挂上权威法律数据库的 MCP 适配器（律所 IT 部门一次配置，全所共享）。如果暂未部署 MCP，可先做退化版本——让 skill 输出"待人工核查清单"，律师拿到清单后人工去 Westlaw 逐条点。
3. **调用**：律师在 Claude Code 里敲 `/cite-check 我的备忘录.md`，5-10 秒后拿到结构化核查报告——红色标记疑似幻觉（不存在）、黄色标记引文页码不符、绿色标记通过。
4. **审计留档**：在 skill 末尾追加一步——把核查报告自动写入 `cite-check-log/YYYY-MM-DD-<案件号>.md`，留作所内合规审计 trail。

**注意事项：** ① **不要让 skill 帮你"修复"幻觉**——只识别、不修改，修改必须人手并标注修改原因；② skill 的 prompt 本身要被 code review——和代码一样，错误的 skill 定义会扩散错误；③ 客户端隐私——MCP 工具链如调用云端 Westlaw API，应走律所企业账号，避免个人订阅泄漏检索关键词；④ 5/5 前可结合 DeepSeek V4-Flash cache hit 价（$0.028/M）跑非涉密的判例库批量预检（不涉客户内容时），成本几乎为零。
