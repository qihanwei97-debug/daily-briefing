### 🔥 头条一：Anthropic 5/6 SF 大会签 SpaceX Colossus 1 算力——220K GPU + 300MW、Claude Code 限额翻倍

5 月 6 日（周三）旧金山，Anthropic 第二届年度开发者大会 **Code with Claude SF 2026** 当日宣布**与 SpaceX 签订 Colossus 1 数据中心全部算力使用协议**——锁定 **超过 220,000 张 NVIDIA GPU + 超过 300 MW 新增容量**，月内全部上线，专供 Claude Code 与 Claude API 业务（[Anthropic 官方公告（fetch 403）](https://www.anthropic.com/news/higher-limits-spacex) | [Inc. 5/6](https://www.inc.com/ben-sherry/anthropic-and-spacex-just-announced-a-colossal-deal-to-supercharge-claude-ai/91341165) | [CoinDesk SpaceX/Colossus 1 ahead of June IPO](https://www.coindesk.com/tech/2026/05/06/anthropic-signs-elon-musk-s-spacex-for-colossus-1-compute-ahead-of-june-ipo) | [Engadget Claude Code 限额翻倍](https://www.engadget.com/2166315/anthropic-is-doubling-claude-code-rate-limits-after-deal-with-spacex/) | [Neowin 算力危机解](https://www.neowin.net/news/anthropic-taps-spacexs-colossus-supercomputer-to-solve-claude-compute-crunch/) | [The Decoder 220K GPU](https://the-decoder.com/anthropic-taps-spacexs-colossus-1-data-center-for-220000-gpus-to-power-claude/) | [Tech Portal 5/7](https://thetechportal.com/2026/05/07/anthropic-secures-massive-spacex-compute-deal-as-it-expands-claude-code-limits) | [Seeking Alpha "rivals turn partners"](https://seekingalpha.com/news/4587198-rivals-turn-partners-as-anthropic-inks-deal-to-secure-computing-power-from-xais-colossus-1)）。**Anthropic 官方页面 anthropic.com/news/higher-limits-spacex 今日 fetch 403，未取得官方原文；下文据 7 家英文媒体交叉报道还原**。

**关键事实（多源交叉）：**

| 维度 | 关键数据 |
|------|----------|
| **算力规模** | >220,000 NVIDIA GPU + >300 MW 新增容量；Anthropic 锁定 Colossus 1 **全部容量**——非分租 |
| **数据中心** | Colossus 1（位于 Memphis，由 xAI 运营、SpaceX 持股；Seeking Alpha 标题"rivals turn partners"——Anthropic 与 xAI 在模型层是直接竞争对手） |
| **上线时间** | 协议公告月内（即 5 月内）逐步交付 |
| **Claude Code 限额变化** | **Pro/Max/Team/Enterprise 5 小时限额翻倍**；Pro/Max **高峰时段限速取消**；Opus 系列 API 限额"显著提升"（具体数字未披露） |
| **背景** | SpaceX 计划 6 月 IPO（CoinDesk 5/6 标题"ahead of June IPO"）——Anthropic 现金支付为 SpaceX IPO 预 booking 收入 |

**Anthropic 算力供应商联盟全景（截至 5/6）：**

| 供应商 | 协议规模 | 主芯片 |
|--------|----------|--------|
| **AWS** | 5 GW + 100B+ 美元（10 年） | Trainium |
| **Google Cloud** | 5 GW + **$200B（5 年）** | TPU（详见头条二） |
| **Microsoft Azure** | $30B 容量 | NVIDIA |
| **SpaceX Colossus 1** | 220K GPU + 300 MW（**新签**） | NVIDIA |
| **Fluidstack** | $50B 投资 | NVIDIA |

**为什么重要：** ① **算力多元化达到结构性程度**——Anthropic 已不再是"单点依赖 AWS"或"单点依赖 Google"，而是横跨 5 家、覆盖 NVIDIA + Trainium + TPU 三大芯片栈；这是 frontier model 厂商首次公开把"算力供应商组合多元化"做成可对账级别的资产负债表披露——同行（OpenAI 主要绑 Microsoft / Oracle，Google DeepMind 自有 TPU，xAI 自建 Colossus）尚无类似规模的多供应商组合；② **Claude Code 限额翻倍 vs MAX 用量回潮**——4 月 Anthropic 多次因 Claude Code 重度用户用量超出预期临时降级限速（Reddit r/ClaudeAI 长期投诉热区），本次翻倍直接回应这一争议；对所内已部署 Claude Code 的同事，**5 月 7 日起同样订阅档位的可用 token 量翻倍**——但这是按"窗口"计算（每 5 小时），不是按月度池化；③ **"竞争对手算力供应"先例**——Colossus 1 是 xAI 的旗舰超级计算机，xAI 的 Grok 模型与 Claude 直接竞争；Anthropic 从中租用算力，意味着即便 frontier 厂商互为竞品，**算力层面的"批发关系"已超越产品层面的竞争**——这给未来 AI 反垄断分析提出新维度：算力市场是否已从基础设施变成"准公共事业"；④ **SpaceX IPO 预 booking 含义**——SpaceX 6 月 IPO 前已锁定 Anthropic 这笔大额营收，对 SpaceX 招股书估值是直接支撑；同时也意味着**Musk 系（xAI + SpaceX）正在用基础设施资产捕获 frontier AI 行业的现金流**——即便 xAI 自身模型市场份额未跻身第一梯队。

### 🔥 头条二：Anthropic 与 Google 签 5 年 $200B TPU/Cloud 大单——单笔吃掉 Google Cloud backlog 40%+

**5 月 5-6 日**多家媒体援引 The Information 独家报道：**Anthropic 已与 Google 签订一份 5 年期、总额约 $200B 的云计算与 TPU 算力承诺合同**——按金额是 Anthropic 最大、按 Google Cloud 客户单独披露 backlog 占比也是史上最高（[The Information 原报](https://www.theinformation.com/articles/anthropic-commits-spending-200-billion-googles-cloud-chips) | [Engadget](https://www.engadget.com/2165585/anthropic-reportedly-agrees-to-pay-google-200-billion-for-chips-and-cloud-access/) | [Tech Portal 5/6](https://thetechportal.com/2026/05/06/anthropic-enters-200-billion-agreement-with-google-for-cloud-and-tpu-chips) | [Idaho Business Review 5/6](https://idahobusinessreview.com/2026/05/06/anthropic-commits-200-billion-google-cloud-chip-capacity/) | [HeyGoTrade 详解](https://www.heygotrade.com/en/news/anthropic-google-200-billion-tpu-deal-samsung-trillion-sandisk/) | [Seeking Alpha](https://seekingalpha.com/news/4586580-anthropic-inks-200b-google-cloud-chip-pact---report) | [Anthropic 官方公告（fetch 403）](https://www.anthropic.com/news/expanding-our-use-of-google-cloud-tpus-and-services)）。**Anthropic 与 The Information 一手页今日 fetch 403，未取得官方原文 / 付费墙原文；下文据 5 家辅助报道交叉**。

**关键数据：**

| 维度 | 数据 |
|------|------|
| **总承诺** | ~$200B（5 年期；4 月签、5/5-5/6 公开）|
| **算力规模** | "多个吉瓦"（multiple gigawatts）TPU 容量 |
| **新容量上线** | 2027 年起逐步交付 |
| **占 Google Cloud backlog** | **>40%**（按 Google 已披露 cloud revenue backlog） |
| **AWS + Microsoft + Google 三家 backlog 中 Anthropic+OpenAI 占比** | **>50%**（Anthropic 占 Google + AWS、OpenAI 占 Microsoft + Oracle）|
| **附带 Google 对 Anthropic 投资** | 累计承诺最高 $40B；首期 $10B 已注入；估值 ~$350B；后续按里程碑追加 |

**为什么重要：** ① **AI 算力市场已"信用卡化"——少数 frontier 厂商单合同就锁定 hyperscaler 五年容量的一半以上**——这意味着 mid-tier 企业客户在 2027-2030 年想批量采购 TPU/GPU 算力会面临**实质性挤出**，要么排队、要么转向 specialized 二线供应商（CoreWeave/Lambda/Nebius/Crusoe）；这与 Big Tech Q1 财报上调 Capex（5/6 晨报头条二）形成完整闭环：**hyperscaler 加 Capex 是为了消化已签长约**，不是机会主义扩产；② **Anthropic 算力组合"AWS + Google + Azure + SpaceX + Fluidstack"全景与 OpenAI 形成对照**——OpenAI 是"Azure + Oracle + Stargate（OpenAI/SoftBank/Oracle JV）"的相对集中，Anthropic 走分散化；这是 AI 投资人估值两家时的核心结构差异——分散降单点风险，集中保议价能力，难以一概而论；③ **$350B Google 投资估值确认**——Anthropic 在 OpenAI（$500B 估值传闻）之后稳坐第二把交椅；按 Anthropic 公开 ARR 测算（$8-10B 年化），P/S 倍数约 35-44x，仍属 frontier 厂商区间；④ **TPU 上量节奏**——Google "multi-GW" 容量 2027 起交付意味着 2026 全年 Anthropic 主要靠 AWS Trainium + 现有 Google TPU + SpaceX Colossus 1 顶住——SpaceX 220K GPU 是"近月"补丁、Google TPU 是"长周期"主干。

## ⑤ 本期技巧：用 Claude × Microsoft 365 单 agent 跨 Excel/Word/PPT/Outlook 上下文打通做合同审阅

**场景：** 你在汉坤负责一个跨境 M&A 项目——客户 brief 在 Outlook 邮件正文，关键交易条款在客户发来的 Excel 财务模型，备忘录草稿在 Word 文档，路演 deck 在 PowerPoint，所有文件都不允许下载离开本地（数据合规）。过去做合同审阅或备忘录起草时，你打开 ChatGPT / Claude，把 Word 段落复制进去问"这一段责任条款合理吗？"——AI 给出的建议常常**与 Excel 里的对价拆分不一致、与 Outlook 邮件中客户实际诉求脱节**——因为它根本没看见 Excel 与 Outlook 的内容，只看到你粘进来的 Word 片段。Anthropic 5 月 5 日纽约金融服务日发布的**Claude × Microsoft 365 完整集成**——尤其是"单 agent 跨 4 个应用携带上下文"——正是为这类场景设计的。

**为什么好用：** 与上期 GPT-5.5 Instant 的"跨会话/文件/Gmail 引用"是两家不同路径——OpenAI 路径是用户主动调用历史会话作搜索结果；Anthropic 路径是**Claude 作为单一 agent 同时驻留在 4 个 Office 应用、自动从应用本身读取当前打开内容**——你不需要复制粘贴，Claude 能直接看到你的 Excel 表当前 sheet、Word 文档当前段落、PPT 当前幻灯片、Outlook 当前邮件线程，并把 4 处上下文合在一起回答。这与 Microsoft 自家 Copilot 最大的差异是**底座是 Claude Opus 4.7（在 Anthropic 路径上）而非 GPT-5.5（在 Copilot 路径上）**——选项变多。对律师工作流的实际增益是：**审合同时不必再人工重述背景**——AI 自动把 Excel 对价拆分、Outlook 客户最新邮件诉求、Word 当前条款、PPT 给客户的展示草稿四处信息融合后再给建议，幻觉与隐式假设错误显著降低。

**操作路径：**

1. **确认所在所或部门已开通 Anthropic Claude Enterprise 账号**——5/5 公告中 MS 365 完整集成在 Enterprise 优先推送，所内 IT 可向 Anthropic 销售或合作伙伴申请部署；Pro / Max 个人账号本期暂不在范围
2. **在 Microsoft 365 应用商店搜索"Claude for Microsoft 365"**（具体上架名以 Anthropic 销售确认为准），按提示授予 Claude 对 Excel / Word / PPT / Outlook 的"读"权限——**不要授予"写"权限**（律师工作流中 AI 直接修改文件风险过高）
3. **启动一个跨应用任务时按固定顺序打开应用**——先打开 Outlook 找到客户最新需求邮件，再打开 Excel 关键 sheet（对价 / 时间表 / 触发条件），再打开 Word 当前正在审的合同条款，最后打开 PPT 路演 deck；Claude 跨应用上下文按"打开顺序 + 当前激活窗口"采集
4. **在任意一个应用中调出 Claude 侧边栏**（具体快捷键以正式发布版为准），输入提问："基于当前 Outlook 邮件中客户的最新诉求 + Excel 当前 sheet 中的对价拆分 + Word 当前段落的责任条款，哪些条款与客户诉求不一致？给出修订建议"——Claude 会同时引用 4 处内容回答
5. **强制人工审核**——AI 给出的修订建议**永远先打印对照原文逐条核对**，**绝不直接接受 AI 建议覆盖客户原文**；建立内部"AI 修订建议 review log"，记录每次接受/拒绝率，月度复盘

**注意事项：** 第一，**MS 365 完整集成在 Enterprise 限定**——Pro/Max/个人账号本期不能用，所内若未签 Enterprise 合同则无路径；第二，**客户敏感数据进入 Claude 的合规风险需所内合规组先评估**——Anthropic Enterprise 合同应明确"客户数据零留存 / 不进入训练 / 区域数据存储"三条款，签前由合规组红线 review；第三，**与 GPT-5.5 Instant 的跨会话引用并存使用**——两家路径不冲突，一家做"跨应用同步上下文"（Anthropic）、一家做"跨会话历史搜索"（OpenAI），律师可按场景选；第四，**5/5 至 5/7 该集成尚在分批推送中**——若所内尚未收到推送，可向 Anthropic 销售提请 priority access。
